# tagforge3d

Утилита для генерации STL-моделей **жетонов** с выпуклой надписью под печать на FDM-принтере с соплом 0.4 мм. Имя строки на входе — STL на выходе, готовый к слайсеру.

Терминология (жетон / пластина / надпись / отверстие) и инварианты зафиксированы в [`CONTEXT.md`](CONTEXT.md). Архитектурные решения — в [`docs/adr/`](docs/adr/).

## Что делает

Берёт строку текста и формирует STL-файл жетона:

- **Пластина** — скруглённый прямоугольник, размеры считаются по содержимому.
- **Надпись** — выпуклые буквы поверх пластины, высота настраивается в слоях печати.
- **Отверстие** — сквозной цилиндрический вырез слева, под кольцо-крепёж.

Левая стенка между отверстием и краем пластины **жёстко равна толщине пластины** — сечение получается квадратное, через колечко продевается легко.

## Ограничения

| Параметр                     | Значение                                                  |
| ---------------------------- | --------------------------------------------------------- |
| Сопло                        | 0.4 мм                                                    |
| Алфавит надписи              | `0-9`, `A-Z`, `a-z`, `+`, `-`, `(`, `)`, `.`, `,`, `_`, `/`, пробел |
| Кириллица                    | не поддерживается                                         |
| Длина строки                 | без жёсткого лимита (целевой кейс — телефон, ~12 символов) |
| Минимальная стенка (по умолчанию) | 0.8 мм (`plate_layers × layer_height` = `4 × 0.2`)   |

Символы вне whitelist'а → ошибка валидации с указанием конкретного нарушающего символа. Утилита не пытается «починить» ввод заменой на `?` или пропуском.

## Параметры (CLI)

| Флаг                     | Тип    | По умолчанию | Описание                                                                                        |
| ------------------------ | ------ | ------------ | ----------------------------------------------------------------------------------------------- |
| `--text`                 | str    | **required** | Содержимое надписи                                                                              |
| `--output`               | path   | `{sanitize(text)}`     | Stem выходных файлов (без расширения). Утилита всегда пишет **два STL**: `{stem}-plate.stl` и `{stem}-inscription.stl` — чтобы в слайсере можно было покрасить надпись и пластину в разные цвета. Дефолт — текст надписи с заменой `/` на `_` (единственный символ из whitelist, запрещённый в FS). Существующие файлы перезаписываются без подтверждения. |
| `--layer-height`         | float  | `0.2`        | Высота слоя печати, мм                                                                          |
| `--inscription-layers`   | int    | `2`          | Толщина надписи в слоях. Итоговая высота = `layer-height × inscription-layers`                  |
| `--plate-layers`         | int    | `4`          | Толщина пластины в слоях. Заодно задаёт ширину левой стенки в мм (`plate-layers × layer-height`) |
| `--font-size`            | float  | `8.0`        | Высота букв, мм                                                                                 |
| `--font`                 | path   | `fonts/SourceCodePro-Bold.ttf` | TTF-шрифт. Выбран эмпирически по результатам тестовой печати (см. ниже).            |
| `--padding`              | float  | `2.0`        | Внутренний зазор между отверстием и текстом, мм. Внешняя рамка пластины не настраивается — она всегда равна `plate_thickness` (см. формулу ниже). |
| `--hole-diameter`        | float  | `3.0`        | Диаметр отверстия под кольцо, мм                                                                |

При значениях по умолчанию надпись выступает на `0.2 × 2 = 0.4` мм над пластиной толщиной `0.2 × 4 = 0.8` мм.

## Геометрия

```
                         plate_width
   ┌────────────────────────────────────────────────────────────┐
   │     ⊙          + 7 1 2 3 4 5 6 7 8 9 0                     │   plate_height
   └────────────────────────────────────────────────────────────┘
    ├──┤←─────→├←────────────────────────────→┤├┤
    ↑   ↑      ↑              ↑                ↑
    │   │      │              │                border (= plate_thickness)
    │   │      padding        text_width
    │   hole_diameter
    border (= plate_thickness, инвариант: левая стенка под кольцо)
```

### Принцип

- **`border`** (внешняя рамка) — со всех 4 сторон равна `plate_thickness`. Инвариант. Не настраивается. Левая стенка такой ширины обеспечивает удобство надевания на кольцо; ту же толщину применяем сверху/снизу/справа — получаем визуально однородный «бортик».
- **`padding`** (внутренний зазор) — только между отверстием и текстом. Это единственный настраиваемый отступ.
- **Высота пластины** измеряется по реальному bbox глифов (`bb.ylen`), а не по em-размеру шрифта. Иначе для текста без descender'ов (цифры, заглавные) пластина получалась бы заметно выше, чем буквы.

### Формулы

```
plate_thickness = plate_layers × layer_height
border          = plate_thickness                                  # все 4 стороны рамки

text_width      = bb.xlen  (измерено по реальным контурам глифов)
text_height     = bb.ylen  (измерено по реальным контурам глифов)

plate_width     = border + hole_diameter + padding + text_width + border
plate_height    = text_height + 2 × border
corner_radius   = padding / 2

hole_center     = (border + hole_diameter / 2,  plate_height / 2)
inscription     = центрирована по Y; левый край = border + hole_diameter + padding
```

Если при заданных параметрах отверстие не помещается по высоте (`hole_diameter > font_size`) — утилита падает с ошибкой валидации, а не «умно» подращивает пластину.

## Технологический стек

- **Python 3.10+** (`match`, современные type-hints).
- **[CadQuery](https://github.com/CadQuery/cadquery)** — параметрическая CAD-генерация, текст из TTF, экспорт STL. См. [ADR-0001](docs/adr/0001-cadquery-as-cad-backend.md).
- **CLI:** `argparse`, без зависимостей.
- **Логирование:** `print()` в stderr, без модуля `logging` (overkill для одного уровня INFO).
- **Упаковка:** `pyproject.toml` + точка входа `python -m tagforge3d`. Без публикации в PyPI.

## Исследование: выбор шрифта по умолчанию

Шрифт выбирался **эмпирически**, печатью пробной плашки. Чисто аналитически не угадать, как штрихи в 0.8 мм лягут на конкретный шрифт.

### Тестовая плашка

Dev-скрипт [`scripts/generate_font_test.py`](scripts/generate_font_test.py) (не часть CLI) генерит **два STL** для multi-material печати:

- `out/font-test-base.stl` — подложка (2 слоя = 0.4 мм)
- `out/font-test-inscriptions.stl` — 6 строк надписей (2 слоя = 0.4 мм, лежат поверх подложки)

В слайсере оба грузятся как одна сборка (через **Add part** или **Merge into single object**), и каждой части назначается свой филамент.

Состав плашки:

```
   ╔══════════════════════════════════════════════════╗
   ║  DejaVu Mono     1234567890 +                    ║
   ║  JetBrains       1234567890 +                    ║
   ║  Source Pro      1234567890 +                    ║
   ║  Inconsolata     1234567890 +                    ║
   ║  Roboto Mono     1234567890 +                    ║
   ║  DejaVu Sans     1234567890 +                    ║ ← пропорциональный, для контроля
   ╚══════════════════════════════════════════════════╝
```

- Размер букв на тесте = боевой `font-size` (8 мм). Иначе тест бессмысленен — толщина штрихов уплывёт.
- Высота надписи на тесте = `inscription_layers × layer_height` = 0.4 мм (как в реальном жетоне).
- Габарит плашки — 131 × 78 × 0.8 мм.

### Результат

Топ-3 после визуальной оценки напечатанной плашки:

1. **Source Code Pro Bold** — «прикольный и аккуратный». **→ принят как дефолт `--font`.**
2. **JetBrains Mono Bold** — «просто аккуратный».
3. **DejaVu Sans Mono Bold** — «строгий».

Inconsolata, Roboto Mono и пропорциональный DejaVu Sans в топ-3 не вошли. TTF-файлы всех шести кандидатов оставлены в `fonts/` — на случай, если позже понадобится сравнить ещё раз или дать пользователю выбор через `--font`.

## Логирование

По умолчанию утилита печатает ход работы в stderr — пользователю не нужно гадать, заглох ли процесс:

```
[validate] text '+71234567890' OK (12 chars, allowlist match)
[font]     loaded DejaVu Sans Bold from fonts/DejaVuSans-Bold.ttf
[measure]  text bounds: 58.4 × 8.0 mm
[layout]   plate: 66.4 × 12.0 × 0.8 mm  (left wall 0.8, padding 2.0)
[geometry] plate base extruded
[geometry] inscription extruded (height 0.4 mm = 2 layers × 0.2)
[geometry] hole cut at (2.3, 6.0) mm, diameter 3.0 mm
[geometry] boolean union complete
[export]   tessellating to STL (linear deflection 0.05 mm)
[export]   wrote keychain.stl (142 KB) in 1.8 s
```

Ошибки идут туда же, без префикса в скобках, с ненулевым кодом возврата. Флаг `--quiet` пока не вводим (premature).

## Использование

```bash
# Имя файлов по умолчанию = текст надписи
python -m tagforge3d --text "+71234567890"
# → +71234567890-plate.stl + +71234567890-inscription.stl

python -m tagforge3d --text "BAG-042" --font-size 6
# → BAG-042-plate.stl + BAG-042-inscription.stl

# Явный стем
python -m tagforge3d --text "+71234567890" --output out/keychain
# → out/keychain-plate.stl + out/keychain-inscription.stl

# Тонкая настройка геометрии
python -m tagforge3d --text "+71234567890" --inscription-layers 3 --hole-diameter 4
```

В слайсере оба STL грузятся как одна сборка: правый клик по объекту → **Add part** → выбрать второй файл. Координаты совпадают, надпись ляжет на пластину точно по месту. Дальше каждой части назначаешь свой филамент.

Если не нужен multi-material — просто открой оба файла в слайсере одной сборкой и печатай одним пластиком.

## Backlog

- **Прочность стенки 0.8 мм.** При толщине пластины в 4 слоя сечение левой стенки получается 0.8 × 0.8 мм — на PLA выдерживает повседневную трепку, но при ударах может треснуть. Если в эксплуатации появятся поломки — поднять дефолт `--plate-layers` до 6.
- **Флаг `--quiet`** — если кому-то окажется шумно от лога по умолчанию.
- **Кириллица** — добавить, если понадобится. Расширить allowlist + проверить, что выбранный шрифт содержит кириллические глифы.
- **Валидация «непустого» текста.** Сейчас whitelist пропускает строку из одних пробелов — формально она «валидна», но даст пустой жетон и имя файла вида `   .stl`. Добавить требование: хотя бы один символ должен быть не-пробельным.
